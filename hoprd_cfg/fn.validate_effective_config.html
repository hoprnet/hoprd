pub(crate) fn validate_effective_config(args: &[String]) -> Result<()>
Expand description

Validates the effective hoprd configuration built from args (a hoprd CLI argument vector, without the leading program name).

This mirrors hoprd’s own startup exactly: the YAML file with env-var and CLI-flag overrides layered on top, so values legitimately supplied via the environment or CLI are not falsely reported as missing or invalid. A --help/--version request is treated as success without validating, since hoprd itself renders those.
